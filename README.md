# Лабораторная 3 — Метод баланса (вариант 1)

C++/CLI WinForms-приложение: решение краевой задачи для ОДУ
$\frac{d}{dx}\left(k(x)\frac{du}{dx}\right) - q(x)u(x) = -f(x)$ на $(0,1)$
с разрывными коэффициентами в точке $\xi=0.4$. Метод однородных
консервативных схем (метод баланса) + прогонка.

Поддерживаются граничные условия трёх родов:
- **Дирихле** (1-го рода): `u = γ`
- **Нейман** (2-го рода): для левого края `−k(0)·u'(0) = γ`, для правого `k(1)·u'(1) = γ`
- **Робин** (3-го рода): на левом краю `−k(0)·u'(0) + σ·u(0) = γ`, на правом `k(1)·u'(1) + σ·u(1) = γ`

## Файлы

- `Solver.h` — численное ядро (нативный C++): метод баланса, прогонка, аналитическое решение тестовой задачи.
- `MainForm.h` — главная форма (C++/CLI WinForms): UI, графики, таблицы.
- `App.cpp` — точка входа.

## Создание проекта в Visual Studio

Тестировалось в VS 2019/2022 с workload’ом «Desktop development with C++» и
установленным «.NET Framework 4.x targeting pack».

### 1. Создать проект

`File → New → Project → CLR Empty Project (.NET Framework)`. Назвать,
например, `Lab2_BalanceMethod`. Если такого шаблона нет — установите
индивидуальный компонент **«C++/CLI support»** через Visual Studio Installer.

### 2. Добавить файлы

Скопировать в папку проекта три файла (`Solver.h`, `MainForm.h`, `App.cpp`)
и добавить их в проект (`Project → Add → Existing Item…`).

**Важно:** файлы должны быть в кодировке **UTF-8 with BOM**, иначе MSVC
прочитает русские строки как cp1251 и в форме будут «кракозябры».
В VS: `File → Save As… → стрелка возле Save → Save with Encoding → Unicode (UTF-8 with signature)`.

### 3. Подключить ссылки на сборки .NET

`Project → Add Reference → Assemblies → Framework`:

- `System`
- `System.Drawing`
- `System.Windows.Forms`
- `System.Windows.Forms.DataVisualization`

Без последней сборки графики не будут компилироваться.

### 4. Настроить свойства проекта

`Project → Properties` (для конфигурации **All Configurations**):

- `Configuration Properties → Advanced → Common Language Runtime Support`: **Common Language Runtime Support (/clr)**
- `Configuration Properties → General → C++ Language Standard`: **ISO C++17 (/std:c++17)** (или новее)
- `Configuration Properties → Linker → System → SubSystem`: **Windows (/SUBSYSTEM:WINDOWS)** *(чтобы не появлялось окно консоли)*
- `Configuration Properties → Linker → Advanced → Entry Point`: **main**

Если не хочется убирать консоль — можно оставить SubSystem по умолчанию (Console),
тогда рядом с формой будет открываться консольное окно (это нормально для отладки).

### 5. Целевой фреймворк

`Project → Properties → Common Properties → .NET Targeting → Target Framework Version`:
**v4.7.2** или новее. C++/CLI стабильно работает с .NET Framework 4.x;
с .NET 5+ /clr пока имеет ограничения, лучше Framework.

### 6. Сборка и запуск

`F5`. Должно появиться окно «Лабораторная 2 — Метод баланса (вариант 1)».

## Использование

1. Задайте `n` (по умолчанию 100).
2. Выберите типы граничных условий слева и справа. Для Робина появится
   поле `σ`. Для тестовой задачи по умолчанию `μ₁ = 0`, `μ₂ = 1` —
   значения γ должны быть теми же, что и в условии.
3. **«Решить тестовую»** — решает тестовую задачу (коэффициенты заморожены
   в значениях $k_1^*=1.4$, $k_2^*=0.4$, $q_1^*=0.4$, $q_2^*=0.16$,
   $f_1^*=0.4$, $f_2^*=e^{-0.4}$), сравнивает численное решение с
   аналитическим, показывает $\varepsilon_1 = \max|u-v|$.
4. **«Решить основную»** — решает основную задачу с разрывными
   коэффициентами на сетках $n$ и $2n$, оценивает $\varepsilon_2 = \max|v-v_2|$
   по правилу Рунге.
5. **«Проверка порядка»** — запускает оба расчёта на $n = 10, 20, 40, 80, 160, 320$,
   показывает отношение $\varepsilon(n)/\varepsilon(2n)$ — для схемы 2-го
   порядка должно быть около 4.
6. **«Широкий диапазон n»** — заполняет таблицу 6 из бланка отчёта
   при $n$ от 10 до 1 000 000 со значениями $-\lg\varepsilon_1$, $-\lg\varepsilon_2$
   для построения графиков сходимости в Excel/Origin.

## Проверка корректности

При $n=320$ для тестовой задачи с дефолтными ГУ ожидаемая
$\varepsilon_1 \sim 10^{-7}$ (схема 2-го порядка, $h \approx 3\cdot 10^{-3}$).
Для основной задачи на той же сетке $\varepsilon_2$ должна быть того же порядка.

## Заметки об особенностях схемы

- **Гармоническое осреднение** $a_i = h / \int_{x_{i-1}}^{x_i} dx/k(x)$ — это
  то, что обеспечивает сохранение порядка 2 даже при разрыве $k(x)$. Все
  интегралы для варианта 1 берутся в замкнутом виде через первообразные
  (`ln(x+1)`, `ln(x)`, `x²/2`, `x³/3`, `−exp(−x)`), численного квадратурного
  шума нет.
- **Граничный узел** обрабатывается интегрированием уравнения по
  полуячейке `[0, h/2]` (или `[1−h/2, 1]`); это даёт второй порядок и в
  Неймане, и в Робине без апроксимаций производных односторонними
  разностями (которые обычно дают порядок 1).
- **Прогонка** имеет диагональное преобладание благодаря $q_1=x>0$ на
  $[0,h/2]$ — даже при чистом Неймане ($\sigma=0$) член $\tilde d_0 > 0$
  обеспечивает устойчивость.
