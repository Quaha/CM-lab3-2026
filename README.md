# Лабораторная работа 3 — Метод баланса (вариант 7)

C++/CLI WinForms-приложение: решение краевой задачи для ОДУ
$\dfrac{d}{dx}\!\left(k(x)\dfrac{du}{dx}\right) - q(x)\,u(x) = -f(x)$
на $(0,1)$ с разрывными коэффициентами в точке $\xi=\pi/4$.
Используется однородная консервативная разностная схема (метод
баланса, интегрально-интерполяционный метод) и метод прогонки.

## Параметры варианта 7

| | $j = 1$ (слева от $\xi$) | $j = 2$ (справа от $\xi$) |
|---|---|---|
| $k_j(x)$ | $\sqrt{2}\,\cos x$ | $2$ |
| $q_j(x)$ | $x$ | $x^{2}$ |
| $f_j(x)$ | $\sin 2x$ | $\sin x$ |

Граничные условия:

- **1-я краевая задача:** $u(0) = \mu_1 = 1$, $\;u(1) = \mu_2 = 1$.
- **3-я краевая задача (теплообмен по Ньютону):**
  $w(0) = -\gamma_1(u(0) - \theta_1)$, $\;w(1) = -\gamma_2(\theta_2 - u(1))$,
  где $w(x) = -k(x)\,u'(x)$ — функция теплового потока,
  $\gamma_1 = 4,\;\gamma_2 = 3,\;\theta_1 = 2,\;\theta_2 = 1$.

Тестовая задача строится по основной заморозкой коэффициентов на пределы в $\xi$:
$k_1^{\*}=1,\;k_2^{\*}=2,\;q_1^{\*}=\pi/4,\;q_2^{\*}=(\pi/4)^2,\;f_1^{\*}=1,\;f_2^{\*}=\sqrt{2}/2$.

## Файлы

- `Solver.h` — численное ядро (нативный C++): метод баланса, аналитическое
  решение тестовой задачи, обе схемы аппроксимации ГУ (классическая и
  улучшенная).
- `Progonka.h` — метод прогонки для трёхдиагональной СЛАУ в форме Самарского.
- `MyForm.h` — главная форма (C++/CLI WinForms): UI, графики, таблицы.
- `MyForm.cpp` — обработчики кнопок.
- `App.cpp` — точка входа.

## UI

Три селектора в верхней панели:

1. **Постановка задачи**: «1-я краевая» / «3-я краевая».
2. **Тип задачи**: «Тестовая» / «Основная».
3. **Схема аппроксимации ГУ**: «Классическая» / «Улучшенная»
   (доступна только при выборе 3-й краевой задачи; для 1-й краевой
   значения в граничных узлах задаются точно, и схема ГУ не нужна).

Три действия:

- **Решить** — рассчитать выбранную конфигурацию.
  Для тестовой задачи показывает $\varepsilon_1 = \max|u(x_i) - v(x_i)|$,
  для основной — $\varepsilon_2 = \max|v(x_i) - v_2(x_{2i})|$ по правилу
  Рунге. На вкладке «Решение» — справка, два графика (решения + разность)
  и таблица из бланка отчёта.
- **Проверка порядка** — расчёты на $n = 10, 20, 40, 80, 160, 320$,
  заполняет таблицы 4 и 5 из бланка отчёта. Для 2-го порядка
  отношение $\varepsilon(n)/\varepsilon(2n) \approx 4$,
  для 1-го — $\approx 2$.
- **Широкий диапазон n** — таблица 6 при $n$ от 10 до 1 000 000
  с колонками $-\lg\varepsilon_1$ и $-\lg\varepsilon_2$ для построения
  графика сходимости.

## Разделение задач по бланку отчёта

В лабораторной четыре подзадачи, по бланку отчёта:

| # | Подзадача | Селекторы |
|---|---|---|
| 1 | 1-я краевая, тестовая | 1-я краевая · Тестовая |
| 2 | 1-я краевая, основная | 1-я краевая · Основная |
| 3 | 3-я краевая, тестовая, классическая аппроксимация | 3-я краевая · Тестовая · Классическая |
| 4 | 3-я краевая, основная, улучшенная аппроксимация | 3-я краевая · Основная · Улучшенная |

Каждый из четырёх членов команды переключает UI на нужную комбинацию и
получает все материалы для своего раздела отчёта (справка, графики,
таблицы, отношения погрешностей).

## Особенности схемы

- **Гармоническое осреднение** $a_i = h / \int_{x_{i-1}}^{x_i} dx/k(x)$ — то,
  что обеспечивает сохранение 2-го порядка сходимости даже при разрыве
  $k(x)$ (Модуль 4, теорема 2 о сходимости схемы).
- **Все интегралы** для коэффициентов $a_i, d_i, \varphi_i$ берутся
  *аналитически* через первообразные, прописанные в `Piecewise`
  (для $\sqrt{2}\cos x$, $2$, $x$, $x^2$, $\sin 2x$, $\sin x$ всё
  выражается через элементарные функции). На участках, пересекающих
  $\xi$, интеграл автоматически разбивается. Численного квадратурного
  шума нет.
- **Классическая аппроксимация ГУ** (3-я краевая, Модуль 7, п. 7.3):
  односторонняя разностная производная на торце. Порядок аппроксимации
  ГУ = 1, схема в целом сходится с порядком 1.
- **Улучшенная аппроксимация ГУ** (3-я краевая, Модуль 7, п. 7.4):
  интегрирование уравнения по полуячейке $[0, h/2]$ методом баланса.
  Порядок аппроксимации ГУ = 2, схема сохраняет 2-й порядок сходимости.
- **Прогонка** в форме Самарского, проверка применимости — Модуль 4,
  теорема о корректности схемы; диагональное преобладание выполняется
  благодаря $q(x) \ge 0$ и $\gamma_1, \gamma_2 > 0$ (для 3-й краевой).

## Сборка в Visual Studio

`File → New → Project → CLR Empty Project (.NET Framework)`. Если нет
шаблона — установить компонент **«C++/CLI support»** через Visual Studio
Installer.

В проект добавить пять файлов: `Solver.h`, `Progonka.h`, `MyForm.h`,
`MyForm.cpp`, `App.cpp`. Файлы должны быть в кодировке **UTF-8 with BOM**,
иначе MSVC прочитает русский текст как cp1251.

Свойства проекта (для **All Configurations**):

- *Configuration Properties → Advanced → Common Language Runtime Support:*
  **Common Language Runtime Support (/clr)**
- *C/C++ → Language → C++ Language Standard:* **ISO C++17 (/std:c++17)** или новее
- *Linker → System → SubSystem:* **Windows (/SUBSYSTEM:WINDOWS)**
- *Linker → Advanced → Entry Point:* **main**
- *Common Properties → .NET Targeting → Target Framework Version:* **v4.7.2** или новее

References (`Project → Add Reference → Assemblies → Framework`):
`System`, `System.Drawing`, `System.Windows.Forms`,
`System.Windows.Forms.DataVisualization`.

## Проверка корректности

Ориентиры для 1-й краевой задачи, тестовая, $n = 320$:
$\varepsilon_1$ должна быть порядка $10^{-6}$..$10^{-7}$ (схема 2-го порядка,
$h \approx 3\cdot 10^{-3}$). Для основной — $\varepsilon_2$ того же порядка.

Для 3-й краевой задачи с классической схемой ГУ — порядок будет 1,
$\varepsilon \propto h$; с улучшенной схемой — порядок 2, $\varepsilon \propto h^2$.
Это видно по столбцу «отношение» на вкладке «Проверка порядка».
